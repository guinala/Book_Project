# Workflow Preferences

## Mostrar código antes de aplicar
Mostrar el código propuesto en el chat antes de aplicarlo, y pedir confirmación ("¿Lo aplico?").

**Why:** El usuario pidió explícitamente ver el código antes de aplicarlo, y en alguna ocasión pidió deshacer un cambio porque no resolvía el problema.

**How to apply:** Para cualquier cambio no trivial, mostrar el diff o código nuevo inline primero.

---

## Explicar conceptos JS/TS con ejemplos del proyecto
El usuario hace preguntas conceptuales detalladas sobre JS/TS (arrow functions, Promise.all, destructuring, etc.).

**Why:** Está aprendiendo activamente mientras trabaja en el proyecto, no solo buscando fixes.

**How to apply:** Explicar los conceptos con ejemplos concretos del código del proyecto.
